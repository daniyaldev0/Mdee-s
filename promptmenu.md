# MDEE'S Website
## Brand Refresh — Phase 2 (Global Colors, Buttons & UI Consistency)

Before coding

1. Read BUSINESS_CONTEXT.md
2. Read ENGINEERING_RULES.md
3. Preserve all existing functionality.
4. Continue from Phase 1.
5. Build ONLY this phase.

---

# OBJECTIVE

Apply the new brand identity consistently across the entire website.

The design system has already been updated.

Now update all interface elements to use the new branding.

Do NOT redesign layouts.

---

# STEP 4 — GLOBAL COLOR APPLICATION

Review the entire website.

Replace the previous brand colors with the new design system.

Update

- Buttons
- Links
- Headings
- Badges
- Price tags
- Decorative accents
- Icons
- Borders
- Dividers
- Hover states
- Active states
- Focus states

Every color should come from the global CSS variables.

Remove any hardcoded brand colors where possible.

---

# COLOR HIERARCHY

Use colors intentionally.

Primary Blue

- Main CTAs
- Links
- Active navigation
- Important headings

Warm Yellow

- Prices
- Highlights
- Small accents
- Promotional labels

Pizza Orange

- Very limited usage
- Hot Deals
- Special Offers
- Promotional badges

White / Light Gray

- Cards
- Backgrounds
- Sections

Dark Gray

- Body text
- Secondary headings

Avoid using multiple accent colors together.

Maintain a clean hierarchy.

---

# STEP 5 — BUTTON SYSTEM

Review every button.

Primary Buttons

- Primary Blue background
- White text
- Consistent height
- Consistent padding
- Consistent border radius
- Smooth hover transition

Secondary Buttons

- White background
- Blue border
- Blue text

Hover

- Slight elevation
- Slight brightness adjustment
- Smooth transition

Buttons should feel modern and premium.

---

# CTA CONSISTENCY

Ensure every call-to-action follows one component system.

Examples

- Order Now
- WhatsApp Order
- Call Now
- View Menu
- Foodpanda

No button should look visually different without purpose.

---

# STEP 6 — REDUCE ORANGE

The previous branding relied heavily on orange.

Update the interface so orange becomes an accent only.

Use orange sparingly for

- Promotional tags
- Deal highlights
- Small decorative elements

Blue should become the dominant interface color.

Yellow should support the branding.

The interface should immediately feel aligned with the new logo.

---

# COMPONENT REVIEW

Review

- Cards
- Deal badges
- Category labels
- Section titles
- Icons
- Navigation
- Footer
- Contact cards

Ensure every component follows the new design language.

---

# CONSISTENCY

Maintain

- Existing spacing
- Existing layouts
- Existing typography
- Existing animations
- Existing responsiveness

Only improve branding consistency.

---

# PERFORMANCE

Reuse existing CSS variables.

Remove duplicate color declarations.

Avoid unnecessary CSS.

Do not increase complexity.

---

# CODE QUALITY

Keep everything

- Modular
- Reusable
- Maintainable
- Production ready

Avoid hardcoded colors whenever possible.

---

# IMPORTANT

Do NOT

- Redesign sections
- Change layouts
- Change typography
- Add new animations
- Modify content
- Introduce new UI components

Only refresh the existing interface.

---

# OUTPUT

Modify only

- style.css
- index.html (only if required)

Do not regenerate unrelated code.

At the end provide

1. Components updated
2. Colors replaced
3. Remaining branding recommendations

The finished website should feel cohesive, modern and professionally branded around the new MDEE'S identity while preserving the existing user experience.
